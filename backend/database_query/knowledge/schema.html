Database Schema Documentation

This file contains the **design overview of the database**, including all tables, their relationships, column data types, and indexing strategy. It will be used to help the AI understand the database structure.

Example Table Documentation

Table: finepayments

  • id (UUID, PK)
  • user_id (FK → users.id)
  • date (DATE)
  • fine_amount (DECIMAL) – Gross fine applied
  • finededuction (DECIMAL) – Deduction rules applied
  • net_amount (DECIMAL) – Computed as fine - deduction

Relationships

finepayments.user_id → users.id (many-to-one). Each payment belongs to a user record.
