Exception Handling Guide

This file contains common errors, ambiguities, or edge cases and how the system should handle them before or after generating SQL.

Rules

Missing Column in Query

If a user asks for a column not found in schema:

  • Check for similar column names (typo correction).
  • If multiple matches: ask clarifying question.
  • If none: return error message with suggestion.

Ambiguous Metric

If query requests "revenue", ask:

  • "Do you mean gross revenue, net revenue, or adjusted revenue?"

Restricted Query Type

- Only SELECT queries allowed. - If user asks for `DELETE`, `INSERT`, or `UPDATE`, respond with: "This system only supports READ queries for safety."

Empty Result

If query returns zero rows:

  • Respond gracefully: "No records found, please refine filters."
  • Suggest potential filters (date range, status, etc.).
